# AI 论文共写工作台当前产品完成情况

更新时间：`2026-07-11`

## 1. 当前结论

当前产品已经完成 `v1.x MVP 主流程 + v1.8 原创实证补强 + v1.9 文献补充增强 + v2.0 全学术人群升级`，并完成 `v2.0.1 学术文档统一工作台首轮收口`。当前阶段是 `章节唯一正文源下的深度可用性打磨`，不再只面向本科课程论文，也不再维护两个并列的正文工作台。

它已经不是单纯原型、文档或半联调阶段，而是具备了可运行的前后端、真实 Supabase PostgreSQL、真实 AI 调用、材料解析、初稿生成、共写预览、审查复查、证据绑定、知识库检索、版本管理和导出能力。

当前更准确的定位是：

| 维度 | 当前判断 |
|---|---|
| 产品主线 | “学术文档”是唯一主工作台；`DocumentSection` 是唯一可编辑正文源，整篇只用于检查、组装与交付 |
| 前端页面 | 支持章节写作 / 整篇检查切换、可拖拽章节树、章节编辑、选择性共写、质量抽屉和只读整篇交付视图 |
| 后端接口 | 可信链、原创风险、审查、复查、共写和导出已迁移到章节 / 文档作用域；旧 draft API 保留兼容 |
| 数据库 | 真实 Supabase 已执行 v2.0 基础迁移、章节版本修复和 v2.0.1 作用域迁移；旧数据继续保留 |
| AI 能力 | Prompt 已改为动态学术画像、文档类型、研究范式和章节上下文，不再硬编码课程论文或本科生表达 |
| Demo 演示 | 可完整走通 |
| 自动化测试 | 后端 service / controller 测试已形成基础体系；前端具备生产构建、MVP smoke 和 Playwright E2E |
| 上线能力 | 暂未进入生产上线准备 |

一句话总结：

`当前项目保留完整的 v1.x 兼容能力，并已形成面向本科、硕士、博士和科研人员、以章节为唯一正文源的证据驱动型研究共创工作台。`

这里的 `MVP 100%` 仅表示 PRD 定义的 MVP 验收范围已经完成，不表示完整产品化或生产上线能力达到 100%。

## 2. 当前产品主流程

当前已锁定的用户主路径为：

```text
新建研究项目与学术画像
-> 上传写作输入
-> 文件预处理 / 文本抽取 / 图片 OCR
-> AI 语义解析
-> 解析质量清单 / 材料补全与分类修正
-> 学校 / 导师 / 课程 / 期刊 / 用户要求基准
-> 当前文档动态 readiness / 文献补充增强入口
-> 构建项目知识库
-> 创建或切换开题、学位论文、论文稿件等多个文档
-> 为当前文档选择材料范围
-> 按章节生成、编辑和共写预览
-> 整版 / 逐段 / 差异行选择性应用为章节新版本
-> 查看本章可信链、原创风险、审查、申诉与复查
-> 在整篇检查中聚合覆盖率、引用一致性和待处理问题
-> 点击问题返回对应章节与正文范围
-> 按章节组装并导出 DOCX / PDF
```

当前产品原则：

| 原则 | 状态 |
|---|---|
| AI 是创作者 / 共写者，不是强制裁判 | 已贯彻 |
| 系统提供流程化引导，但不强制用户逐步完成所有任务 | 已贯彻 |
| 材料不足时不兜底生成，而是提示补充材料并提供真实文献检索入口 | 已实现 v1.9 |
| 候选文献只是待下载线索，必须上传原文并完成 AI 解析后才参与生成 | 已实现 v1.9 |
| 关键材料必须完成 AI 解析后才进入生成链路 | 已实现 |
| 审查结果可申诉，不把 AI 误判作为唯一结论 | 已实现 |
| 知识库只纳入已解析材料，避免污染证据链 | 已实现 |
| AI 修改默认先预览，用户确认后才应用 | 已实现 |
| 可信链重建走异步 job，避免页面长时间卡死 | 已实现 |
| 解析结果不可靠时先提示补充，不替用户兜底生成 | 已实现 v1.5 |
| 正文交付前提示可信链、引用、原创实证和 AI 写作味风险 | 已实现 v1.8 |
| 不承诺规避检测，而是提示空泛无据段落并引导补真实案例 / 数据 / 来源 | 已实现 v1.8 |
| 研究计划不要求已有结果，系统综述不要求实验数据，学位论文按范式和章节检查研究成果 | 已实现 v2.0 |
| 一个研究项目可包含多个独立文档并共享材料和知识库 | 已实现 v2.0 |
| 章节生成与知识检索只使用当前文档选定材料 | 已实现 v2.0 |
| AI 操作记录材料依据、模型、采纳状态和披露要求 | 已实现 v2.0 |
| `DocumentSection` 是唯一可编辑正文源，整篇视图不可直接改全文 | 已实现 v2.0.1 |
| 章节保存只自动重建可信链和本地风险，不自动调用 AI 深度审查 | 已实现 v2.0.1 |
| 章节版本变化后旧检查结果标记为 `STALE`，不冒充当前结论 | 已实现 v2.0.1 |
| 旧整篇工作台从导航隐藏，路由和 API 只保留历史兼容 | 已实现 v2.0.1 |

## 9. 当前最重要的产品缺陷收口情况

### 9.1 可信链可验证性已完成一轮收口

当前已经把段落级证据链迁入章节 / 文档作用域，并补充覆盖率、确认率、缺来源统计、五步链路卡、原文定位、原始材料位置线索和证据强弱解释。系统按章节版本与段落指纹判断检查是否仍然有效；用户可以从证据卡定位正文范围，并在整篇检查中查看聚合覆盖率。

已完成链路：

```text
正文段落
-> 章节版本与段落指纹
-> 关键论点
-> 支撑证据片段
-> 原始材料位置
-> 页码 / 知识库片段 / 摘录级定位
-> 文献信息
-> 正文引用
-> 定位正文
-> 证据强弱解释
```

后续只剩更深层增强：PDF 页码级精准跳转、原文截图定位、证据强弱的更精细规则。

### 9.2 共写预览可控性已完成一轮收口

当前已经把“先预览，后应用”迁入章节工作台，并新增选区共写、逐句新增/删除差异、高亮对照、修改理由、保留约束检查和选择性应用。用户可按整版、段落或差异行创建一个章节新版本；如果预览后正文已变化，系统返回 `409`，不会覆盖新修改。

已完成能力：

| 方向 | 说明 |
|---|---|
| 逐句 diff | 已完成新增/删除句级差异展示 |
| 高亮 diff | 已完成候选正文 / 当前正文的新增删除高亮 |
| 修改理由 | 已完成每条差异的原因提示 |
| 约束检查 | 已完成引用、数据、新增文献、作者表达约束提示 |
| 局部接受 | 已完成按差异行或段落选择性创建章节新版本 |
| 整版应用 | 已保留应用为章节新版本 |
| 版本冲突保护 | 已完成预览基准版本校验，冲突返回 `409` |

后续只剩更深层增强：词级 diff、差异合并冲突处理和更细的回滚体验。

### 9.3 审查复查闭环已完成一轮收口

当前支持章节 / 整篇手动 AI 审查和单条复查，并已在抽屉里补充复查依据、下一步建议与历史。审查刷新按问题指纹更新而不是删除历史；章节共写应用后，关联项自动进入“已修改待复查”，章节版本变化后的旧分析会标记为过期。

已完成能力：

| 方向 | 说明 |
|---|---|
| 复查历史 | 已新增 `review_recheck_logs` 持久化 |
| 历史展示 | 已在审查抽屉展示最近复查记录 |
| 复查依据 | 已展示依据清单和依据快照 |
| 状态闭环 | 已展示状态变化和影响级别变化 |
| 共写关联 | 已在共写预览中提示可能关联的审查项，并保存到 `cowrite_preview_review_links` |
| 章节共写关联 | 已保存到 `section_cowrite_preview_review_links`，应用后进入 `MODIFIED_PENDING_RECHECK` |
