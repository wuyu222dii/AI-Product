## 2. 当前产品主流程

当前已锁定的用户主路径为：

```text
新建项目
-> 上传写作输入
-> 文件预处理 / 文本抽取 / 图片 OCR
-> AI 语义解析
-> 解析质量清单 / 材料补全与分类修正
-> 老师要求快照
-> 材料充足性检查 / 文献补充增强入口
-> 生成初稿
-> 构建项目知识库
-> 进入共写工作台
-> 查看材料可信链
-> 查看原创实证与 AI 写作味风险
-> 查看覆盖率评分 / 引用一致性 / 原始材料预览入口
-> 生成 AI 共写预览
-> 用户逐段接受或确认后应用为新版本
-> 审查与手动复查
-> 版本管理
-> 导出 DOCX / PDF 并查看交付风险提示
```

当前产品原则：

| 原则 | 状态 |
|---|---|
| AI 是创作者 / 共写者，不是强制裁判 | 已贯彻 |
| 系统提供流程化引导，但不强制用户逐步完成所有任务 | 已贯彻 |
| 材料不足时不兜底生成，而是提示补充材料并提供真实文献检索入口 | 已实现 v1.9 |
| 候选文献只是待下载线索，必须上传原文并完成 AI 解析后才参与生成 | 已实现 v1.9 |
| 关键材料必须完成 AI 解析后才进入生成链路 | 已实现 |
| 审查结果可申诉，不把 AI 误判作为唯一结论 | 已实现 |
| 知识库只纳入已解析材料，避免污染证据链 | 已实现 |
| AI 修改默认先预览，用户确认后才应用 | 已实现 |
| 可信链重建走异步 job，避免页面长时间卡死 | 已实现 |
| 解析结果不可靠时先提示补充，不替用户兜底生成 | 已实现 v1.5 |
| 正文交付前提示可信链、引用、原创实证和 AI 写作味风险 | 已实现 v1.8 |
| 不承诺规避检测，而是提示空泛无据段落并引导补真实案例 / 数据 / 来源 | 已实现 v1.8 |
