# AI 论文共写工作台 v2.0 全学术人群规格

更新时间：`2026-07-11`

## 1. 产品定位

`AI 论文共写工作台` 是面向本科生、硕士生、博士生与科研人员的证据驱动型研究共创平台。

AI 默认角色是研究共创助手：可以基于用户确认的真实材料生成、修改和审查，但不能编造数据、实验、访谈、伦理审批、引用或原创贡献，也不能替代导师、评审或作者做最终学术判断。

## 2. 核心对象

```text
研究项目 Workspace
├── 学术项目画像 AcademicProjectProfile
├── 共享材料与项目知识库
└── 多个学术文档 AcademicDocument
    ├── 文档专属材料范围
    ├── 章节树 DocumentSection
    ├── 章节版本 DocumentSectionVersion
    ├── 章节级 AI 生成 / 共写预览
    └── AI 使用记录 AiActionLog
```

一个研究项目可以同时包含开题、学位论文、期刊稿和研究报告。材料默认项目共享，但用户可以为每个文档建立独立材料范围。

## 3. 学术画像

| 维度 | 当前支持 |
| --- | --- |
| 学术阶段 | 本科、硕士、博士、科研人员 |
| 学科组 | 理工、医学健康、社会科学、人文、商法、交叉学科 |
| 研究范式 | 定量、定性、混合、实验、计算、设计科学、理论、系统综述、其他 |
| AI 策略 | 仅指导审查、基于证据生成、允许完整生成但强制人工复核 |
| 其他 | 主要语言、学校/机构、默认引用格式 |

## 4. 文档类型

当前支持课程论文、研究计划/开题、本科毕业论文、硕士论文、博士论文、期刊论文稿、会议论文稿、文献综述和研究报告。

创建文档时系统按文档类型与研究范式生成默认章节树。用户可新增自定义章节，并通过鼠标、触控或键盘拖拽整理章节顺序；排序只调整结构，不创建新的正文内容版本。

## 5. 动态 Readiness

readiness 不是统一的课程论文门槛，而是由 `文档类型 + 研究范式 + 当前章节` 决定。

- 研究计划不要求已有研究结果，但需要研究问题、文献和方法依据。
- 定量或实验型学位论文的结果、讨论章节需要真实数据、样本或分析结果。
- 定性研究需要访谈、观察、田野记录或编码依据。
- 计算与设计科学需要代码说明、模型输出、测试记录或原型验证。
- 系统综述不要求实验数据，但需要检索、筛选和证据综合材料。
- 博士论文增加原创贡献确认；该检查只提示，不替代学术评审。
- 文档整体未就绪时，不受缺口影响的章节仍可手动编辑；章节级 readiness 通过后也可调用 AI。

## 6. 要求优先级

```text
用户上传并确认的学校 / 导师 / 课程 / 期刊要求
> 当前文档专属设置
> 研究范式规则
> 平台默认模板
```

项目级要求与文档级要求分别保存。AI prompt 不再硬编码课程论文、本科生或老师要求。

## 7. 章节共写

- AI 生成只作用于当前章节。
- 共写默认先生成预览，不直接覆盖正文。
- 用户可以设置改写深度、保留引用、保留数据、不新增来源和保留作者声音。
- 应用预览后生成新章节版本，旧版本可恢复。
- 文档组装按章节顺序输出，并汇总章节来源追溯。
- AI 操作记录作用文档、章节、材料依据、模型、输出摘要、采纳状态和披露要求。

## 8. 材料与知识库

- `MaterialCategory` 保留兼容。
- 新增 `materialRole` 与 `researchArtifactType`，覆盖文献、提交要求、作者草稿、研究成果、方法协议、数据集、分析输出、访谈、问卷、伦理文件、评审意见等。
- 知识库仍为项目级构建，但检索可传 `documentId / materialIds / tags`，只返回当前文档允许使用的材料片段。
- v2.0 使用 PostgreSQL 全文检索、关键词和结构化筛选，不使用 embedding。

## 9. 数据库与兼容

正式迁移：[20260711031857_v2_academic_workspace.sql](../../supabase/migrations/20260711031857_v2_academic_workspace.sql)

版本一致性修复：[20260711065932_repair_document_section_versions.sql](../../supabase/migrations/20260711065932_repair_document_section_versions.sql)

- 旧 workspace 自动映射为 `UNDERGRADUATE + COURSE_PAPER`。
- 旧当前 draft 回填为主文档的 `正文` 章节，正文与来源追溯不丢失。
- 新规则和追溯字段使用 JSONB，稳定状态使用受约束 text。
- 所有外键和章节查询建立索引；章节与知识片段建立全文检索 GIN 索引。
- 新表启用 RLS；当前仍由 Spring Boot 使用数据库连接访问，不向 Supabase Data API 主动授权。
- Runtime `ddl-auto` 默认关闭，数据库变更只通过迁移执行。
- 远端 Supabase migration history 已登记 `20260711031857 / 20260711065932`，本地与远端版本一致。
- 章节写入以章节表和历史版本表的最大版本为基准；AI 处理期间章节发生变化时返回冲突，不覆盖用户的新修改。

## 10. 当前边界与下一步

v2.0 已完成首轮工程闭环和主流程 Playwright E2E，但尚未完成生产级权限、多人协作、对象存储、持久化队列、监控、CSL、RIS/BibTeX、LaTeX、期刊投稿深度规则、组件级测试与更多异常路径 E2E。

推荐下一步：

1. v2.1 学位论文深度版：研究问题、方法、结果、讨论与证据一致性。
2. CSL + RIS/BibTeX + 正式学位论文模板与 LaTeX。
3. AI 使用披露与 CRediT 贡献角色。
4. 导师/合作者批注、权限隔离和多人协作。
