## 一、软件工程与系统开发方向（适合 CS732/SOFTENG 732 课程）

### 题目 1：基于物联网与机器学习的高校教室能源智能管理系统设计与实现

**英文题目**：Design and Implementation of an Intelligent Classroom Energy Management System Based on IoT and Machine Learning

#### 研究背景与意义

- **现实背景**：全球高校能源消耗中，教室照明与空调占比高达 40%-60%，传统人工控制方式存在严重的能源浪费。奥克兰大学承诺 2030 年实现碳中和，但现有教室能源管理仍以人工为主，效率低下。
- **理论意义**：丰富物联网技术在建筑能源管理领域的应用研究，提出适用于高校特殊场景的能源优化算法。
- **实践意义**：为奥克兰大学及新西兰其他高校提供可复制、可推广的能源管理解决方案，助力碳中和目标实现。

#### 国内外研究现状

- 国外：欧美高校已广泛应用物联网能源管理系统，但多采用简单的定时控制策略，缺乏对用户行为和教室占用的动态预测。
- 国内：清华大学、浙江大学等已开发基于机器学习的能源管理系统，但在新西兰气候条件和高校管理模式下的适用性有待验证。
- **研究缺口**：现有研究很少结合教室使用时间表、环境传感器数据和用户偏好进行综合优化，且缺乏针对新西兰高校的成本效益分析。

#### 核心研究内容与关键问题

1. 教室能源消耗特征与影响因素分析
2. 基于 LSTM 神经网络的教室占用率预测模型构建
3. 结合用户偏好的自适应能源控制算法设计
4. 智能能源管理系统的开发与集成
5. 系统性能测试与成本效益评估

**关键问题**：如何在保证用户舒适度的前提下最大化能源节约？如何解决传感器数据噪声对预测精度的影响？

#### 研究方法与技术路线

- **研究方法**：文献研究法、实验法、对比分析法、成本效益分析法
- **技术路线**：
  1. 需求分析与数据收集（奥克兰大学 401 号楼 3 个月能源数据）
  2. 预测模型训练与算法优化
  3. 系统架构设计（前端：React 19，后端：Spring Boot，通信：MQTT）
  4. 原型系统开发与部署
  5. 现场测试与效果验证
  6. 论文撰写与修改

#### 论文结构安排

1. 绪论
2. 相关技术与理论基础
3. 系统需求分析与总体设计
4. 能源预测模型与控制算法设计
5. 系统详细设计与实现
6. 系统测试与结果分析
7. 结论与展望

#### 预期研究成果与创新点

- **预期成果**：
  1. 开发一套可部署的智能教室能源管理原型系统
  2. 发表 1 篇学术会议论文（可选）
  3. 提交一份完整的毕业论文
- **创新点**：
  1. 提出了一种结合 LSTM 预测模型和模糊控制的自适应能源管理算法
  2. 首次在新西兰高校环境下验证了基于机器学习的能源管理系统的有效性
  3. 构建了包含技术、经济和环境三个维度的综合评估体系

#### 可行性分析

- **技术可行性**：你已掌握 React、Spring Boot 和 MQTT 协议的使用，机器学习模型可使用 TensorFlow 或 PyTorch 实现，技术成熟。
- **数据可行性**：可从奥克兰大学设施管理部门获取能源消耗数据，教室使用时间表可从学校官网获取。
- **时间可行性**：整个研究周期约 12-16 周，符合毕业论文时间要求。
